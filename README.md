# SoSandbox 使用文档

## 1. 软件简介

SoSandbox 用于对 Android Native so 文件进行查看、分析与修改。

当前版本架构支持说明：

- 目前仅支持 `ARM64` 架构分析与相关执行能力
- 非 `ARM64` 目标在部分功能下可能无法正常工作
- 涉及执行类能力时，需在支持环境下使用

当前版本已实现的主要能力包括：

- 打开并解析 so 文件（支持文件选择器、外部应用分享打开）
- 查看函数列表
- 查看字符串表
- 查看 PLT 表
- 查看 RegisterNatives 结果
- 函数反汇编查看（支持长按选中文本复制）
- 伪 C 查看
- Hex 查看与修改
- 汇编指令修改
- 流程图查看
- 交叉引用分析
- 保存修改后的 so 文件
- AI 伪 C 配置与调用（支持 OpenAI / Gemini / 自定义接口）
- 检测更新
- 卡密激活赞助功能

## 2. 打开 so 文件

### 操作步骤

1. 打开软件首页
2. 点击 `选择SO文件`
3. 在系统文件选择器中选择目标 so 文件，或从外部应用（文件管理器、分享）用 SoSandbox 打开
4. 等待解析完成（加载期间会显示 loading 弹窗）

### 解析完成后可执行的操作

- 点击 `函数` 查看函数列表
- 点击 `字符串` 查看字符串列表
- 点击 `PLT` 查看导入跳板信息
- 点击 `RegisterNatives` 查看 JNI 注册信息
- 所有加载过的 so 文件会自动保存到「最近项目」，方便快速重新打开

## 3. 首页功能说明

### 3.1 函数

点击 `函数` 后进入函数列表页面。

可用功能：

- 查看全部函数
- 搜索函数名
- 对函数列表进行排序
- 点击函数进入详细分析页
- 长按函数查看交叉引用
- 手动识别地址为函数：右上角「更多」→「识别地址为函数」，可输入任意地址（支持可执行段内任意位置），并可选手动指定函数大小

### 3.2 字符串

点击 `字符串` 后进入字符串页面。

可用功能：

- 查看 so 中解析出的字符串
- 搜索字符串
- 过滤有价值字符串
- 点击字符串查看其引用位置

### 3.3 PLT

点击 `PLT` 后进入 PLT 页面。

可用功能：

- 查看导入函数跳转表
- 识别外部调用入口

### 3.4 RegisterNatives

点击 `RegisterNatives` 后进入 JNI 注册分析页面。

当前可用内容：

- 静态注册结果查看
- 已激活赞助时可获取动态注册内容

说明：

- 未激活时会显示静态注册的可用结果
- 激活赞助后可使用动态注册能力

## 4. 函数详细分析页

点击函数列表中的任意函数后，会进入函数详细分析页。

当前页面包含以下标签页：

- `汇编`
- `伪C`
- `Hex`
- `流程图`

切换标签页时只能通过顶部 Tab 点击切换（已禁用页面滑动，防止看代码时误触）

## 5. 汇编页

汇编页用于查看与修改函数汇编。

### 支持的操作

- 查看函数反汇编结果
- 点击函数名跳转到目标函数
- 查看当前函数的交叉引用
- 修改指定汇编指令
- 刷新显示修改结果
- **长按汇编代码行可进入文本选择模式，复制任意指令**

### 交叉引用

在汇编页中可查看：

- 哪些函数调用了当前函数
- 当前函数调用了哪些函数

## 6. 伪C页

伪 C 页用于查看反编译结果。

### 当前支持两种模式

- 本地伪 C
- AI 伪 C

### 本地伪 C

默认会先显示本地伪 C 结果。

### AI 伪 C

如在设置中开启 `AI伪C`，软件会在本地伪 C 的基础上继续请求 AI 生成更易读的伪 C。

使用 AI 伪 C 前需要先在设置中配置 AI 参数。

**普通模式（推荐）**：选择服务商（OpenAI / Gemini），填写 API Key 和模型名即可，软件会自动使用内置模板和响应解析。

**高级模式**：如需接入第三方转发接口或自定义请求格式，可切换高级模式，手动填写：
- **API 完整地址**：如 `https://api.openai.com/v1/chat/completions`
- **请求模板**：完整 JSON 请求体，支持占位符 `{{url}}`、`{{key}}`、`{{prompt_raw}}`（body 为对象时使用）、`{{prompt}}`（body 为字符串 JSON 时使用）
- **响应解析路径**：如 `choices[0].message.content`，留空则自动识别 OpenAI / Gemini 格式

如果未配置完整，伪 C 页面会显示对应错误信息。

## 7. Hex 页

Hex 页用于查看和修改函数原始字节。

### 支持操作

- 查看函数十六进制内容
- 直接修改字节
- 修改后同步到保存逻辑

## 8. 文本页

文本页用于以纯文本方式查看函数内容，适合快速复制或简单浏览。

## 9. 流程图页

流程图页用于查看函数控制流图。

### 说明

- 流程图为赞助功能
- 未激活时不可用
- 激活后会从服务端获取配置并生成流程图

如果服务端未授权或配置获取失败，页面会提示先激活赞助。

## 10. 交叉引用分析

当前交叉引用功能可直接使用。

### 可用入口

- 在函数列表中长按函数
- 在汇编页中查看函数交叉引用
- 在字符串页中点击字符串查看引用

### 可查看内容

- 调用当前函数的来源
- 当前函数调用到的目标
- 某个字符串被哪些函数引用

点击交叉引用结果中的条目后，可以直接跳转到对应函数。

## 11. 修改与保存 so 文件

当前支持以下修改方式：

- 在汇编页修改指令
- 在 Hex 页修改字节

### 保存方式

在函数详细页点击右上角保存按钮即可保存修改。

### 返回时提示

如果存在未保存修改，退出函数详细页时会提示：

- 保存
- 不保存
- 取消

## 12. 设置页面说明

设置页面当前可用功能如下。

### 12.1 夜间模式

可切换深色/浅色主题。

### 12.2 AI 设置

用于配置 AI 伪 C 所需参数。

**普通模式**：选择服务商、填写 API Key 和模型名即可，软件自动处理请求模板和响应解析。

**高级模式**：手动配置完整请求体和响应解析路径，适合自定义接口或中转服务。

### 12.3 伪C提示词

可自定义发送给 AI 的伪 C 提示词内容。

### 12.4 函数发现方式

SO 文件解析时识别函数的策略，提供三种模式：

- **精准（仅符号表，最快）**：只使用 ELF 符号表中的函数符号，速度最快，但无法识别被剥离符号的函数（如 libil2cpp.so）
- **平衡（符号表+轻量扫描，推荐）**：符号表 + init/fini 数组 + 所有可执行段的轻量扫描，兼顾速度与覆盖率，适合大多数情况
- **最大覆盖（全段扫描，函数最多）**：增加 ADRP 入口识别、降低 BL 调用阈值、扫描所有可执行段，函数最多但可能有少量误报，耗时明显更长

切换后需重新加载 SO 文件才能生效。

### 12.5 代码字体大小

可调整代码类页面的字体大小。

### 12.6 赞助激活

可输入卡密激活赞助功能。

激活后可使用的内容包括：

- 动态 RegisterNatives
- 流程图

点击 `检测更新` 后，软件会从 GitHub Release 检查是否有新版本。

### 12.9 关于软件

显示当前版本和软件简介。

## 14. 使用建议

### 建议一：先查看函数再进入详细分析

推荐流程：

1. 打开 so
2. 进入函数列表
3. 搜索目标函数
4. 进入详细页查看汇编、伪 C、Hex、流程图

### 建议二：修改前保留原始文件

虽然软件支持保存修改，但仍建议你先备份原始 so 文件。

### 建议三：AI 功能需要你自己提供接口

AI 伪 C 不内置第三方模型服务，使用前请自行准备兼容接口和 Key。

## 15. 常见问题

### 15.1 为什么流程图打不开？

流程图是赞助功能，未激活时不可用。

### 15.2 为什么 AI 伪 C 失败？

通常是以下原因：

- 未配置 API Key
- 网络超时
- 服务商接口返回格式异常

**注意**：普通模式下软件会自动使用内置模板和响应解析，无需手动填写请求模板和返回路径。只有高级模式才需要手动配置这些参数。

### 15.3 为什么退出时提示有未保存修改？

说明你在汇编页或 Hex 页中做了修改，但还没有保存。

### 15.4 字符串点击后显示引用数量少或没有？

字符串引用需要先扫描函数，首次扫描可能需要一点时间。
